# Semantic HTML + CSS Assignment

## Objective

Structure raw content using **semantic HTML elements only**, then style it using
**CSS that targets semantic elements only**.

---

## Part 1 — HTML Structure

You are given:

```
content.md
```

### Your task:

- Convert the raw Markdown content into a complete HTML document.
- Use **semantic HTML elements wherever appropriate**.
- Do not use generic containers (`div`, `span`) unless absolutely unavoidable.
- Do not use classes or IDs.
- The structure must be meaningful and accessible.

You are expected to use elements such as:

```
<header>
<footer>
<main>
<article>
<section>
<nav>
<aside>
<figure>
<figcaption>
<details>
<summary>
<time>
<mark>
<data>
<address>
<h1>–<h6>
<hgroup>
<dialog>
```

---

## Part 2 — CSS Styling

After completing the HTML:

- Create a separate stylesheet.
- Style the page using **only semantic selectors**.
- Do NOT use:
  - Classes
  - IDs
  - Inline styles

- Focus on:
  - Clean layout
  - Good typography
  - Readability
  - Logical spacing

---

## Deliverables

Submit:

1. `index.html`
2. `styles.css`

Both must validate and render cleanly in a browser.
